# 1. 작업 회고
## 수행 작업 목록
### GIT
- upstream 등록
- 로컬 브랜치로 fetch 및 rebase

### 웹 서버 미션 4단계
- PR 작성 및 Review 반영
  - 지원하지 않는 표준 HTTP 메서드에 대해 405 응답을 반환하도록 수정
  - 헤더를 파싱할 때 대소문자를 구분하지 않는 맵에 담도록 수정

### 조별 활동
- 그룹 세션

### 웹 서버 미션 5단계
- 쿠키 파싱 로직 추가
- 세션 관리자 도입
- 개별 컨트롤러에서 발생하는 ‘에러 응답 생성’ 및 ‘HTTP 메서드 검증’ 책임 분리

## 작업 과정에서 어려웠던 점 & 배운 점
### 신중하게 행동하기
금일 새벽에 이전에 작성했던 PR들이 Merge되었다는 메일을 받았다.

드디어 upstream에 있는 브랜치를 fetch하여 rebase할 수 있다는 생각에, 현재 작업 중인 브랜치를 확인하지 않고 대뜸 fetch 명령을 수행해 버렸다.

평소에 사용하지 않던 명령어들이라 '혹시 문제가 발생했을지도 모른다'는 생각에 급히 해당 명령어들에 대해 알아보았고, 다행히 문제가 없는 상태임을 확인하여 한시름 놓을 수 있었다.

하지만 학습 과정이고 문제가 없었으니 망정이지, 실무에서 꼼꼼히 확인하지 않아 문제가 발생한 상황이었다면 굉장히 아찔했을 것이다.

이 경험을 통해, Git과 같이 공용으로 사용하는 툴의 경우에는 더욱 신중하게 작업하고, 평소에 사용 방법을 숙지해 놓을 필요가 있다는 생각을 하게 되었다.

# 2. 종합 회고
## 잘한 점
### 이해할 수 있게 설명하기
- 지난 회고에서 '이해할 수 있게 설명해야 한다'고 했었는데, 그룹 세션에서 이를 잘 적용했던 것 같다.
- 지속적으로 조원들의 반응을 살피며 내용을 설명하니, 나와 조원 모두 구현 코드에 대한 이해가 깊어져 더 많은 의견을 나눌 수 있었다.

## 아쉬웠던 점
### 퇴근이 평소보다 오래 걸렸다
- 퇴근 길에 열차를 놓쳐, 20분 가량을 허비하게 되었다.

## 개선할 점
### 출퇴근 시간을 줄여보자
- 출퇴근 길은 할 수 있는 작업이 제한적이므로, 가능한 출퇴근 시간을 짧게 가져가는 것이 좋다.
- 환승 시간을 확인하는 등 출퇴근 시간을 단축하려 노력해 보자.

# 3. 학습 내용
## GIT
### 명령어
- fetch: 원격 저장소의 내용을 로컬에 가져오기만 한다.
- rebase: 이름에서 알 수 있듯, 브랜치의 base를 재설정한다.

## 네트워크
### 쿠키 
- 대소문자를 구분한다.
- 단, 서버에서 클라이언트한테 쿠키를 발급할 때 사용하는 속성(SameSite, HttpOnly 등)은 대소문자를 구분하지 않는다.

## 추가적으로 학습할 내용
- String Constant Pool vs Runtime Constant Pool
- 정적(static) 클래스
